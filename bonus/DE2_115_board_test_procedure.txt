DE2-115 Bonus CPU Bring-Up Procedure
====================================

Project files
- Project: CPU_TEST_Sim_DE2_115.qpf
- Revision: CPU_TEST_Sim_DE2_115
- Top-level (compile): CPU_TEST_Sim.vhd -> CPU_TEST_Sim_rtl (+ cpu_run_clock, board_hex_out)
- Block diagram: CPU_TEST_Sim.bdf (legacy flat schematic; see banner in file)
- Memory image: system_memory.mif (Problem 2 default — for-loop / power product)
- Alternate image: system_memory_problem1.mif (Problem 1 if/else-if/else; copy over system_memory.mif before compile)

1) Build/Compile
----------------
From the bonus folder run:
  quartus_sh --flow compile CPU_TEST_Sim_DE2_115

Pass conditions:
- No missing source file errors
- No duplicate top-level/entity errors
- No unconstrained critical I/O for:
  CLOCK_50, KEY[3..0], SW[17..0], HEX0..HEX7, LEDG, LEDR[9..8], LCD_*
- MIF load reported for system_memory.mif

2) Program Board
----------------
- Open Quartus Programmer
- Select generated .sof from output_files/
- Program the DE2-115 board

3) Initial Hardware Sanity
--------------------------
- Ensure board power is stable
- Set SW[0] = 0 initially (CPU stopped)
- Press/release KEY[0] once:
  * KEY is active-low on board
  * Design converts KEY[0] into active-high reset internally
  * LCD controller is always enabled in this revision

Expected:
- HEX0 shows numeric T_info (0..7); HEX1..HEX5 show blank (0); HEX6/HEX7 show PC[7:0] as two hex digits
- LEDR[9..8] and LEDG[7..0] are not stuck
- LCD backlight/control active (LCD_ON high, LCD_EN toggling)

4) Execution Checks (Problem 2 — current default program)
-----------------------------------------------------------
This build runs Problem 2: a=1, then fifteen ROL (one-bit left shift) steps equivalent to
  for(i=1; i<6; i++) a = a * 2^i
Final A = 0x00008000 (32768 = 2^15). Program halts with JMP @ word address 0x10.

- With SW[0] = 0, confirm CPU state is held (HEX/LCD stable; PC on HEX6/HEX7 stops advancing)
- Set SW[0] = 1 to run the CPU on a slow divided clock (~2 Hz) for visible stepping
- Observe LCD line 1 (A) step through shifts until it reaches 0x00008000, then hold
- LCD line 2 (B) is unchanged unless the program writes B (this image does not)
- When halted, HEX6/HEX7 low byte of PC should show 0x10

5) Bonus Program Demonstration Flow
-----------------------------------

Problem 2 (default — system_memory.mif)
- Recompile as above, program board, reset (KEY[0]), run (SW[0]=1)
- Expected signature: LCD line 1 final A = 0x00008000; PC low byte 0x10 at halt
- Repeat reset-run three times; pass if A and PC signature match every time

Problem 1 (alternate — system_memory_problem1.mif)
- Copy system_memory_problem1.mif over system_memory.mif (or replace contents), recompile, reprogram
- Flow: LDAI a, LDBI b, SUB, BEQ branch1, BGU branch2, JMP continue. BGU opcode nibble 0xB.
- Default demo (a=5, b=3): final A = 0x00002222. a=4,b=4 -> 0x1111; a=2,b=9 -> 0x3333 (edit words 0x00/0x01)
- Halt merge at PC 0x1C (self-JMP)

General demo sequence (both problems):
  a) Assert reset with KEY[0] press
  b) Release reset
  c) Set SW[0] = 1 to run / observe
  d) Capture final LCD/HEX signature

6) Stability Check
------------------
- Repeat reset-run cycle three times for each problem image
- Pass if final result signature is identical across all runs
